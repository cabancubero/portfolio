I am building a portfolio to showcase my skills to potential employers and provide documentation for peers/learners. My
  portfolio website has a toggle between "employer" and "learner" modes - code snippets stay the same but                   
  comments/annotations toggle on or off based on the audience.
                                                                                                                            
  For this project, I need you to produce a markdown file at the root with the following sections:

  Context

  If applicable, explain the project's context (e.g., demo app, prototype, production, academic project) and how the
  architecture reflects that context.

  Skills Demonstrated

  A table with two columns: Skill (language, framework, tool, or technique) and Where It Appears (specific location/pattern
  in the codebase). Aim for 6-9 entries.

  Architectural Decisions

  3 decisions that shaped the project. Each should name the pattern, explain what problem it solves, and why this approach
  was chosen over alternatives. Include a short code block if it makes the decision concrete.

  Code Snippets (Core Competencies)

  3-5 self-contained code blocks that each demonstrate a specific, nameable skill. Each snippet should:
  - Have a title and a "Skills:" line listing what it demonstrates
  - Include a 1-2 sentence description of what the code does and why it matters
  - Be self-contained enough to read without surrounding context
  - Map to a skill an employer would recognize on a resume

  Clever Implementations

  1-2 code blocks that go beyond the straightforward solution - custom patterns, elegant problem-solving, or interesting
  abstractions. Each should explain what makes the approach non-obvious and why it works better than the simple alternative.

  ---
  Instructions:
  - Read all source files before selecting snippets. Do not propose code you haven't read.
  - Choose snippets that are diverse - don't repeat the same skill across multiple blocks.
  - Write descriptions that work for both audiences: concise enough for an employer scanning quickly, detailed enough for a
  learner studying the pattern.
  - Do not fabricate or embellish - only highlight what's actually in the code.
  - Name the output file [project_name]_portfolio.md at the project root.
